## 背景
通过[小红书开放平台](https://ad-market.xiaohongshu.com/)可以接入小红书提供的`API`，实现三方客服这样的功能，前提是公司有相关资质。

如果没有资质怎么办呢？可以选择找一家有资质的公司合作，让他们把`API`中转过来。

这样做的好处就是门槛低，不需要资质了。

坏处那就多了，首先是数据都走第三方过了一遍，没什么隐私可言，然后呢需要第三方的配合才能顺利的进行调试开发。

没有资质又想做，也就只能如此了。

## API

小红书提供了的能力如下：
- 广告主授权绑定账户通知三方客服
- 广告主授权解绑账户通知三方客服
- 广告主授权绑定KOS账户通知三方客服
- 查询广告主授权绑定的KOS账户
- 三方客服-发送消息接口
- 三方客服-接收消息接口
- 三方客服-查物料信息接口（落地页/名片/留资卡/笔记/意向评论/交易卡/）
- 意向评论推送
- 三方客服-留资数据回传
- 三方客服-留资和广告归因数据推送

## 踩过坑

### 发送图片

> 先说结论：不要把图片存在七牛云或者自己的服务器，小红书不接收（应该是配置了白名单），经过测试可以放到阿里云，但是不要使用自定义域名，使用默认的阿里云域名

通过[三方客服-发送消息接口]发送图片给用户时，一直失败，反复确认文档没有发现异常

接口中携带的图片是存放在七牛云的，虽然是私有空间，但是携带了签名，确实是可以访问的

反复尝试无果后询问第三方公司（此时怀疑是他们设置了白名单之类的）

沟通确认无白名单相关机制，又尝试了阿里云的图片，仍然不行

于是直接去小红书找了一张图片用来发送，发送成功

说明参数结构是没有问题的，搜索引擎和AI都说阿里云是可以的，于是乎想到是否和自定义域名有关

经过测试确认在使用阿里云默认域名的时候，图片可以正常发送

### 发送视频

> 先说结论：接口返回`message={'code': 10001, 'success': False, 'msg': '5cd5913d0000000010003b33.64290a610000000011020f3b.1ea5897cb0e7f5b', 'data': None}`时不代表发送失败，大概率是发送成功了，只是处理机制是异步进行的，返回一个10001告诉你已经收到了

有了刚才发送图片的经验，心想这发送视频岂不是手拿把掐的事情，结果发送视频`demo`脚本一跑一个失败

开始反思是否是发送的视频编码问题，于是乎从日志里抓了一下之前通过手机发送过来的视频，也就是小红书处理之后的视频和封面

再次运行，还是失败，直接就宕机了，不知道发生了何事

手机打开小红书，发现已经收到了刚才发送的视频

小红书的AI对这种情况的解释是这样的
```markdown
- ​​风控或内容审核延迟​​：
小红书对视频内容有严格的风控和审核机制。审核通过后消息才会真正送达，但接口可能先返回了一个中间状态码（如 10001），而不是最终的发送结果。  
- ​​错误码含义被误解​​：
code: 10001 可能并不是真正的错误码，而是小红书内部的一个​​通用受理码​​或​​异步任务 ID 前缀​​，你的代码可能把它误解析为失败状态。
```
